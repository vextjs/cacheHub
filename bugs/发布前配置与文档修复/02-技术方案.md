#### 依据

`src/redis-adapter.ts` 源码验证：

- `PATTERN_MAX_LENGTH = 512` 常量仅用于 `_toRedisPattern()` 方法（约束 A10）

- SHA-256 压缩逻辑仅存在于 `src/function-cache.ts` 的 `buildCacheKey()`（约束 A09，阈值 1024）

### 3. `package.json`（W01 + W02 + W05）

#### W02 — 顶层 main / types 字段

```diff
+  "main": "./dist/cjs/index.js",
+  "types": "./dist/types/index.d.ts",
```

**说明**：现代工具链通过 `exports` 字段解析，无需顶层字段。但以下场景仍依赖顶层字段：
- TypeScript `moduleResolution: node`（旧项目常见）
- Webpack 4 及以下
- 部分 IDE 插件（不解析 `exports` 条件）

顶层字段指向 CJS 入口（最广泛兼容），与 `exports["."].require` 保持一致。

#### W05 — files 添加 CHANGELOG.md

```diff
  "files": [
-   "dist"
+   "dist",
+   "CHANGELOG.md"
  ],
```

**说明**：`npm publish` 打包 tarball 时仅包含 `files` 数组中声明的路径。`CHANGELOG.md` 为用户了解版本变更的重要文档，profile/05-发布规范.md 要求包含在发布包中。`README.md` 和 `LICENSE` 由 npm 默认包含，无需显式声明。

#### W01 — prepublishOnly 脚本

```diff
+   "prepublishOnly": "npm run typecheck && npm run test:coverage && npm run build",
```

**说明**：`prepublishOnly` 在 `npm publish` 执行前自动触发（`npm pack` 不触发），形成三重安全门禁：
1. `typecheck` — 类型检查，确保无 TS 编译错误
2. `test:coverage` — 全量测试 + 覆盖率，确保无回归
3. `build` — 重新生成 dist 产物，确保发布的是最新构建

执行顺序为串行（`&&`），任一失败则终止发布。

### 4. `src/index.ts`（W03）

```diff
  // 核心实现
  export { MemoryCache } from "./memory-cache.js";
+ export type { SetOptions } from "./memory-cache.js";
```

#### 依据

`src/memory-cache.ts` 中定义：

```typescript
export interface SetOptions {
  tags?: string[];
}
```

`MemoryCache.set()` 方法签名：

```typescript
set(key: string, value: any, ttl?: number, options?: SetOptions): void
```

`SetOptions` 是 `MemoryCache` 标签写入功能的公开 API 参数类型，消费者在使用 `enableTags: true` 时必须能导入此类型。

仅导出 `type`（`export type`），不影响运行时 bundle，符合架构约束 A01（零运行时依赖原则延伸）。
