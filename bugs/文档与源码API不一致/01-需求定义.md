**修复于**: 会话19（2026-03-22）fix 工作流  
**严重程度**: 🔴 高（13 项严重问题直接导致用户代码出错）  

---

## 问题描述

`website/` 文档站中的 API 文档（`api-reference.md` 和 `getting-started.md`）与实际源码实现存在系统性不一致，涉及类型签名、构造参数形式、默认值、字段名等多个维度。

用户若按文档示例编写代码，将遇到以下问题：
- TypeScript 编译错误（字段名不存在、类型不匹配）
- 运行时行为与预期不符（数据未写入、缓存未命中、stats 全为 undefined）
- 分布式缓存失效完全失效（参数形式和接口均错误）

## 根因分析

会话16 自动生成文档时，API 描述未与实际源码实现进行交叉验证，导致以下模块的文档与实现脱节：

1. **MemoryCache** — 构造参数字段名(`ttl`→`defaultTtl`)、默认值、方法返回类型
2. **CacheLike 接口** — `getMany`/`setMany` 使用 `Map` 类型，实际为 `Record`
3. **DistributedCacheInvalidator** — 构造参数(`watchedCaches`→`cache`)、`invalidate` 接口、`InvalidatorStats` 字段名全部错误
4. **FunctionCache** — 构造函数签名、`execute`/`invalidate` 调用形式（可变参数 vs 数组）
5. **MultiLevelCache** — `publish` 回调签名错误
6. **stableStringify** — `undefined` 序列化行为描述错误

文档中所有 API 签名、默认值、字段名、代码示例均与 `src/*.ts` 源码一致，用户可直接复制示例代码运行。
