# 技术方案 — 文档与源码 API 不一致修复

**Bug 编号**: BUG-文档-001  
**优先级**: 高  
**影响范围**: website/docs（不涉及源码逻辑）  
**创建日期**: 2026-03-22  

---

## 根因分析

会话16 生成文档站时，API 描述内容未与实际源码实现进行交叉验证，导致以下系统性偏差：

1. 字段名拼写错误（`ttl` → `defaultTtl`）
2. 默认值抄写错误（`maxEntries: Infinity` 实为 `10000`）
3. 返回类型描述错误（`Map` 实为 `Record`，`void` 实有返回值）
4. 整个模块接口描述错误（`DistributedCacheInvalidator`、`FunctionCache`）
5. 占位符 URL 未替换

### website/docs/guide/api-reference.md

#### MemoryCacheOptions 表格（4 项）

| # | 参数 | 变更内容 |
|---|------|---------|
| B01 | `ttl` | 字段名改为 `defaultTtl` |
| B02 | `maxEntries` | 默认值 `Infinity` → `10000` |
| B03 | `enableStats` | 默认值 `false` → `true` |
| W01 | `maxMemory` | 默认值 `Infinity` → `0`，补充说明 `0 = 无内存限制` |

#### MemoryCache 方法表格（5 项）

| # | 方法 | 变更内容 |
|---|------|---------|
| W03a | `del` | 返回类型 `void` → `boolean` |
| B04 | `getMany` | 返回类型 `Map<string,V>` → `Record<string,V>`，说明同步更新 |
| B05 | `setMany` | 入参 `Map<string,V>` → `Record<string,V>`，返回类型 `void` → `boolean` |
| W03b | `delMany` | 返回类型 `void` → `number` |
| W03c | `delPattern` | 返回类型 `void` → `number` |

#### CacheLike 接口代码块（同步所有签名）

- `del`: `void` → `boolean | Promise<boolean>`
- `getMany`: `Map<string,V>` → `Record<string,V>`
- `setMany`: `Map<string,V>` → `Record<string,V>`，返回 `boolean`
- `delMany`: `void` → `number | Promise<number>`
- `delPattern`: `void` → `number | Promise<number>`

#### CacheStats 类型定义（W02）

新增两个字段：
- `entries: number` — 当前存活条目数
- `memoryUsageMB: number` — 内存占用 MB（保留 3 位小数）

#### MultiLevelCacheOptions 表格（B12）

- `publish` 回调签名：`(keys: string[]) => void` → `(msg: { type: string; pattern: string; ts: number }) => void`
- 触发时机说明：从"写入/删除后调用"改为"`delPattern` 时调用"

#### DistributedInvalidatorOptions 表格（B06 + B13）

- `watchedCaches: CacheLike[]`（可选）→ `cache: CacheLike`（必填）
- `channel` 默认值：`'cache-hub:invalidation'` → `'cache-hub:invalidate'`

#### DistributedCacheInvalidator 方法表格（B07）

- `invalidate` 签名：`(keys: string[])` → `(pattern: string)`

#### InvalidatorStats 类型定义（B08）

完整替换所有字段：

| 旧字段 | 新字段 |
|--------|--------|
| `published` | `messagesSent` |
| `received` | `messagesReceived` |
| `selfFiltered` | `invalidationsTriggered` |
| （缺失）| `errors` |
| （缺失）| `instanceId` |
| （缺失）| `channel` |

#### FunctionCache 章节（B09 + B10 + B11）

- 构造函数：`new FunctionCache({ cache, ttl })` → `new FunctionCache(cache, { ttl })`
- `execute` 签名：`(name, args)` → `(name, ...args)`
- `invalidate` 签名：`(name, args?)` → `(name, ...args?)`
- `getStats` 返回类型：`Record<string, CacheStats>` → `FunctionCacheStats | Record<string, FunctionCacheStats>`

#### stableStringify 特殊值表（W04）

- `undefined` 输出：`'null'` → `'undefined'`，说明改为"避免与 null 缓存键碰撞"
