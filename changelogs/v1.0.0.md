## 📋 变更概述

cache-hub v1.0.0 首个正式发布版本：零运行时依赖的 Node.js 多层缓存库，通过统一的 `CacheLike` 接口整合本地内存缓存、Redis 远端缓存、多级联动、函数装饰器与分布式失效广播。

---

## 🎯 核心变更

### 1. 核心类型与接口（`cache-hub`）

定义统一缓存契约 `CacheLike` 接口，覆盖 11 个必填方法（`get / set / del / exists / has / clear / keys / getMany / setMany / delMany / delPattern`）及 5 个可选扩展方法（`invalidateByTag? / getStats? / resetStats? / destroy? / setLockManager?`）。

新增 `CacheStats` 类型（9 个字段：`hits / misses / sets / deletes / evictions / memoryUsage / memoryUsageMB / entries / hitRate`）与 `MemoryCacheOptions` 配置类型。

### 2. MemoryCache — 内存缓存引擎

基于 ES6 `Map` 实现 O(1) LRU 淘汰，支持 TTL 惰性过期与周期清理、双重容量限制（条目数 + 内存字节）、批量操作、通配符模式删除、统计信息、标签索引（`enableTags`）与全局 `enabled` 开关。

### 3. stableStringify — 稳定序列化

`stableStringify(value, options?)` 生成确定性缓存键：对象键字母排序、`NaN` 固定输出 `"__NaN__"`、循环引用输出 `"[CIRCULAR]"`、`RegExp` / `Date` 特殊处理、`customSerializer` 插件钩子支持自定义类型（如 BSON ObjectId）。

### 4. readThrough — 读穿缓存

`readThrough(cache, ttlMs, key, fetcher)` 实现缓存命中直返、未命中执行 fetcher 并写缓存；内置并发去重（相同 key 共享 Promise）、溢出保护（INFLIGHT_MAX_SIZE = 10000）与超时防漏（300s）。

### 5. MultiLevelCache — 多级缓存

L1（本地）+ L2（远端，可选）双层联动。写策略支持 `'both'`（同步双写）与 `'local-first-async-remote'`（本地优先异步写远端）；远端命中自动回填 L1（`backfillOnRemoteHit`，默认开启）；远端超时降级保护（`remoteTimeoutMs`，默认 50ms）；可选分布式失效广播回调（`publish`，`delPattern` 时触发）。

### 6. RedisCacheAdapter — Redis 适配器

`createRedisCacheAdapter(urlOrInstance)` 将 ioredis 包装为 `CacheLike`。支持 URL 字符串初始化或传入已有实例；`delPattern / keys` 使用 SCAN 游标迭代（禁止 `KEYS` 命令）；`pattern` 超过 512 字符时截断并打印 `console.warn`（key 无长度限制）；JSON 序列化存储，支持 `null` 作为有效缓存值；`close()` 仅关闭自己创建的连接。

### 7. FunctionCache / withCache — 函数缓存

`withCache(fn, options)` 装饰器自动缓存异步函数返回值，键生成规则 `namespace:fnName:stableStringify(args)`（超过 1024 字节自动 SHA-256 压缩）；内置并发去重与条件缓存（`condition`）。`FunctionCache` 类支持 `register / execute / invalidate / invalidatePattern / getStats / resetStats / list / clear`，适合多函数统一管理，支持 per-function TTL / keyBuilder / condition 覆盖。

### 8. DistributedCacheInvalidator — 分布式失效

基于 Redis Pub/Sub 的跨实例缓存失效广播，自动过滤自身消息（`instanceId` 隔离）；`invalidate(pattern)` 广播，接收方对本地缓存执行 `delPattern`；支持可选 `logger` 接入 pino / winston 等；`getStats()` 返回 `messagesSent / messagesReceived / invalidationsTriggered / errors / instanceId / channel`。

## 📋 详细变更清单

### ✨ Added（新增）

#### 核心类型与接口（`cache-hub`）
- 新增 `CacheLike` 接口：定义统一缓存契约，11 个必填方法 + 5 个可选扩展方法（`invalidateByTag? / getStats? / resetStats? / destroy? / setLockManager?`）
- 新增 `CacheStats` 类型：统一统计信息结构（`hits / misses / sets / deletes / evictions / memoryUsage / memoryUsageMB / entries / hitRate`）
- 新增 `MemoryCacheOptions` 配置类型
- 新增 `SetOptions` 类型（标签写入扩展参数，从主入口导出）

#### MemoryCache（`cache-hub`）
- 基于 ES6 `Map` 实现的 O(1) LRU 淘汰引擎
- TTL 支持：惰性过期（`get` 时判断）+ 可选周期清理（`cleanupInterval`）
- 双重容量限制：`maxEntries`（条目数）+ `maxMemory`（字节估算）
- 批量操作：`getMany / setMany / delMany`
- 模式删除：`delPattern(pattern)`（`*` 通配符 → 正则）
- 统计信息：`hits / misses / evictions / hitRate` 等，可通过 `enableStats` 开关
- 标签索引：`enableTags=true` 时维护 `tagIndex`，支持 `invalidateByTag(tag)`
- `enabled` 开关：`false` 时 `get` 返回 `undefined`，`set` 不写入
- `destroy()`：清理周期定时器并清空缓存

#### stableStringify（`cache-hub/stringify`）
- `stableStringify(value, options?)`：对象键字母排序、`NaN` 固定输出 `"__NaN__"`、循环引用输出 `"[CIRCULAR]"`
- `RegExp` / `Date` 特殊处理，数组保序
- `customSerializer` 插件钩子：支持 BSON ObjectId 等自定义类型扩展

#### readThrough（`cache-hub/read-through`）
- `readThrough(cache, ttlMs, key, fetcher)`：缓存命中直返，未命中执行 fetcher 并写缓存
- 并发去重（in-flight map）：相同 key 的并发请求共享同一 Promise
- `ttl ≤ 0` 时直接执行 fetcher 不写缓存
- 内置溢出保护（INFLIGHT_MAX_SIZE = 10000，超限清理最旧 10%）
- 内置超时清理（INFLIGHT_TIMEOUT_MS = 300000ms，防止内存泄漏）
- `undefined` 不写缓存，`null` 视为有效空值

#### MultiLevelCache（`cache-hub/multi-level`）
- L1（本地）+ L2（远端，**可选**）双层缓存，均接受 `CacheLike` 接口
- 写策略：`'both'`（同步双写）/ `'local-first-async-remote'`（本地优先异步写远端）
- 远端命中回填本地（`backfillOnRemoteHit`，默认开启，可关闭）
- 远端操作超时保护（`remoteTimeoutMs`，默认 50ms，超时降级不报错）
- 可选分布式失效广播回调（`publish?: (msg: { type: string; pattern: string; ts: number }) => void`，`delPattern` 时触发）

#### RedisCacheAdapter（`cache-hub/redis`）
- `createRedisCacheAdapter(urlOrInstance)`：将 ioredis 实例包装为 `CacheLike`
- 支持 URL 字符串初始化（自动创建 ioredis 连接）或传入已有实例
- `delPattern / keys` 使用 SCAN 游标迭代，禁止 `KEYS` 命令（防阻塞）
- `close()`：仅关闭自己创建的连接（外部传入的连接不关闭）
- `pattern` 超过 512 字符时截断并打印 `console.warn`（key 无长度限制，不做哈希压缩）
- JSON 序列化存储，支持 `null` 作为有效缓存值
- `getRedisInstance()`：获取底层 ioredis 实例

#### FunctionCache / withCache（`cache-hub/function-cache`）
- `withCache(fn, options)`：装饰器，自动缓存异步函数返回值
- 键生成：`namespace:fnName:stableStringify(args)`，超长键（> 1024 字节）自动 SHA-256 压缩
- 并发去重：相同参数的并发调用共享同一 Promise（实例级 in-flight）
- 条件缓存：`condition(result)` 返回 `false` 时不写缓存
- `FunctionCache` 类：支持 `register / execute / invalidate / invalidatePattern / list / getStats / resetStats / clear`，适合多函数统一管理
- Per-function TTL / keyBuilder / condition 配置，可覆盖全局默认值

#### DistributedCacheInvalidator（`cache-hub/distributed`）
- `DistributedCacheInvalidator`：基于 Redis Pub/Sub 的跨实例缓存失效广播
- 自动过滤自身发出的消息（`instanceId` 隔离）
- 接收失效消息时操作的缓存实例（`cache: CacheLike`，必填单实例）
- 支持 `redisUrl` 字符串或已有 ioredis 连接两种初始化方式
- `invalidate(pattern)` / `close()` 生命周期管理（`pattern` 支持 `*` 通配符）
- 可选 `logger` 选项：接入任意兼容 `debug / info / warn / error` 接口的日志库
- `getStats()` 返回 `messagesSent / messagesReceived / invalidationsTriggered / errors / instanceId / channel`

#### 构建（Build）
- ESM + CJS 双格式产出（`dist/esm/` + `dist/cjs/` + `dist/types/`）
- 多入口按需导入：`cache-hub`、`cache-hub/redis`、`cache-hub/multi-level`、`cache-hub/function-cache`、`cache-hub/distributed`、`cache-hub/stringify`、`cache-hub/read-through`
- CJS 产物兼容性修补：`import.meta.url` → `__filename`（`scripts/build-cjs.mjs`）
- 零运行时依赖（`dependencies: {}`），ioredis 为可选 peerDependency（`>=5.0.0`）
- `prepublishOnly` 安全门禁：typecheck + test:coverage + build

#### 测试（Tests）

- 30 个集成测试（`test/integration/redis.integration.test.ts`），需本地 Redis 连接
- 支持 `REDIS_URL` 环境变量自定义 Redis 地址，`SKIP_INTEGRATION=true` 跳过集成测试

#### 文档站（Website）
- Rspress 文档站：首页（`docs/index.md`）、快速开始（`docs/guide/getting-started.md`）、API 参考（`docs/guide/api-reference.md`）
