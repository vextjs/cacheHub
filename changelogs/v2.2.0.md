# v2.2.0 变更详情

> **发布日期**: 2026-06-03
> **类型**: Minor release
> **前置版本**: v2.1.0

---

## 变更概述

cache-hub v2.2.0 补齐短 TTL 响应缓存所需的 Redis tag 与 lease 底座能力。该版本继续保持 `dependencies` 为空，Redis 仍为 optional peer dependency，并兼容现有三参数 `CacheLike.set` 调用。

---

## Added

- 为 `createRedisCacheAdapter` 新增 Redis tag index，支持 `set(..., { tags })`、`invalidateByTag(tag)`、可配置 `metaKeyPrefix`、`scanCount` 和 `deleteCommand`。
- 新增多级缓存与分布式 tag 失效，包含 `MultiLevelCache.invalidateByTag(tag)` 与 `DistributedCacheInvalidator.invalidateTag(tag)`。
- 新增 `cache-hub/lease` 子入口与 `createRedisLeaseStore` Redis lease 原语。
- 新增 `readThroughWithLease`，用于跨进程读穿缓存 single-flight。
- 补充 tag 失效、Redis lease ownership 与 10,000 请求并发场景的单元测试和 Redis 集成测试。

---

## Changed

- `CacheLike.set` 新增可选第四参数 `CacheSetOptions`，同时保持现有三参数调用兼容。
- Redis pattern 与 tag 删除可通过 `deleteCommand: 'unlink'` 选择 `UNLINK`。
- tag publish workflow 在 `npm publish` 前新增 Redis 集成测试。
- 英文 README 与中文文档补充 tag 失效、Redis lease 用法和完整配置表。

---

## Fixed

- 带 tag 的 key 被不带 tag 的新值覆盖时，会清理旧 tag 元数据，避免旧 tag 误删新的无 tag 值。

---

## Compatibility

- 现有 `CacheLike` 实现和使用三参数 `set(key, value, ttl)` 的消费者保持兼容。
- `dependencies` 继续为空；`ioredis` 仍为 optional peer dependency。
- Node.js 支持下限保持 `>=18.0.0`。
