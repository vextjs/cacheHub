# v2.0.0 变更详情

> **发布日期**: 2026-06-01  
> **类型**: Major release  
> **前置版本**: v1.0.3

---

## 变更概述

cache-hub v2.0.0 聚焦性能优化、Node.js 支持策略更新、限流专用原语和国际化文档入口。本版本将 Node.js 支持下限调整为 `>=18.0.0`，新增 `cache-hub/rate-limit` 可选导出，并补齐 benchmark、Redis 真实集成验证和 npm package metadata。

---

## Breaking Changes

- Node.js 支持下限调整为 `>=18.0.0`。仍使用 Node.js 16 的消费项目需要先升级运行时，再接入 v2.0.0。

---

## Added

- 新增 `cache-hub/rate-limit` 可选导出，提供内存与 Redis 固定窗口限流存储原语。
- 新增 `npm run benchmark`，支持表格输出、`--json` 机器可读输出和 `--output` 文件写入，用于性能基线对比。
- 新增 `docs/README.zh-CN.md` 中文文档，与默认英文 README 互相链接。

---

## Changed

- 优化 `MemoryCache` 批量操作、`withCache` 默认键生成和 `MultiLevelCache` 远端 timeout 清理路径。
- README 默认语言调整为英文，补齐 Redis 默认连接、限流原语、benchmark 与排错说明。
- 补齐 npm package metadata：`description`、`keywords`、`license`、`homepage`、`bugs`、`sideEffects` 与包内 `docs` 文件。
- 将 `ioredis` 作为 devDependency 安装，用于仓库自身 Redis 集成测试；对消费者仍保持 optional peer dependency。

---

## Fixed

- 修复 Redis 集成测试因 optional peer 未安装而只跳过不真实验证的问题；本仓库现在可在本地 Redis 默认无密码连接下执行真实集成测试。

---

## 性能摘要

基于本地 benchmark 与性能优化报告，核心热点路径获得明显提升：

| 场景 | 结果 |
|---|---:|
| `MemoryCache#get hit stats off` | +131.2% |
| `MemoryCache#set existing stats off` | +167.5% |
| `MemoryCache#getMany 20 hits` | +229.3% |
| `MemoryCache#setMany 20 existing` | +174.9% |
| `readThrough cache hit` | +81.6% |
| `withCache hit single key` | +290.8% |
| `MultiLevelCache L1 hit` | +51.3% |
| `MultiLevelCache L1 miss + remote miss` | +197.7% |

> 性能数字来自本地库热点 benchmark，不能替代生产 HTTP middleware 或真实 Redis 网络链路压测。
