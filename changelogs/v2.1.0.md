# v2.1.0 变更详情

> **发布日期**: 2026-06-01
> **类型**: Minor release
> **前置版本**: v2.0.0

---

## 变更概述

cache-hub v2.1.0 补齐高并发原子状态后端能力，用于后续 `flex-rate-limit` 以可选方式复用 `cache-hub` 的 Redis 原子存储层。新增能力不会修改 `CacheLike` 必需接口，也不会新增运行时依赖。

---

## Added

- 新增 `cache-hub/atomic` 子入口，提供 `MemoryAtomicStateBackend`、`RedisAtomicStateBackend` 和对应工厂函数。
- 新增 `AtomicStateBackend`、`AtomicCounterResult`、`RedisAtomicStateClient` 公开类型。
- 扩展 `cache-hub/rate-limit`，新增 `MemoryRateLimitStateStore` / `RedisRateLimitStateStore`。
- 新增 sliding-window、token-bucket、leaky-bucket 状态原语与 rollback token 支持。
- 新增 Redis 真实集成测试：100 并发、10,000 次 `incrementWithTtl` 不丢计数。

---

## Changed

- fixed-window 限流存储内部改为复用通用原子后端。
- README 与中文文档同步补充 `cache-hub/atomic` 和扩展限流状态原语说明。
- package exports 增加 `./atomic`，保持 ESM / CJS / types 三格式一致。

---

## Compatibility

- `CacheLike` 必需接口保持不变。
- `dependencies` 继续保持空依赖，`ioredis` 仍为 optional peer dependency。
- Node.js 支持下限保持 `>=18.0.0`。
