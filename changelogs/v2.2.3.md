# v2.2.3 变更详情

> **发布日期**: 2026-06-11
> **类型**: Patch release
> **前置版本**: v2.2.2

---

## Fixed

- 为 `MemoryAtomicStateBackend` 增加惰性过期回收与 `cleanupExpired()`，避免高基数短 TTL counter 在长进程内持续留存。
- 为 `MemoryFixedWindowRateLimitStore` 与 `MemoryRateLimitStateStore` 增加内存状态回收，覆盖 fixed-window、sliding-window、token-bucket 和 leaky-bucket。
- `DistributedCacheInvalidator.close()` 现在会移除本实例注册的 pub/sub `error` 与 `message` listener，避免复用外部 pub 连接时保留旧闭包。
- Redis tag metadata 增加生命周期收口：带 TTL 的 `key-tags` 反向索引同步 TTL，新增可选 `pruneTagMetadata(tag)`，并在 tagged write 路径执行有界 stale metadata prune。

---

## Changed

- README 与中文文档补充 `cleanupExpired()`、`pruneTagMetadata()` 以及 Redis tag metadata 生命周期说明。
- 新增回归测试覆盖高基数过期/中性状态回收、listener 解绑、metadata TTL 与 bounded prune。

---

## Compatibility

- 无 breaking changes。
- `CacheLike` 必需接口、package exports、构造参数和既有方法签名均保持不变。
- 新增清理能力均为 concrete 或 optional 扩展；消费方无需实现新的必需方法。
- Node.js 支持下限保持 `>=18.0.0`。
- `dependencies` 继续为空；`ioredis` 仍为 optional peer dependency。
