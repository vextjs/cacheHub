# v1.0.3 变更详情

> **发布日期**: 2026-05-30
> **类型**: Patch release
> **前置版本**: v1.0.2

---

## 变更概述

cache-hub v1.0.3 聚焦缓存语义修复、文档入口收口和发布产物精简。该版本修复多级缓存 TTL 回填、分布式失效、函数缓存精确失效和 Redis 批量 key 校验相关问题，同时移除未部署且与 README 高度重复的 Rspress website，并关闭发布构建 source map 输出。

---

## Fixed

- 修复 `MultiLevelCache` 在 L2 命中回填 L1 时丢失剩余 TTL 的问题，避免短 TTL 远端数据被扩展成本地长 TTL 或永久缓存。
- 修复 `DistributedCacheInvalidator.invalidate()` 只广播不先清发送方本地缓存的语义偏差，并同步 README 示例。
- 修复 `MultiLevelCache` 遇到不支持 TTL 查询的远端 `CacheLike` 时不回填 L1 的兼容性回归；TTL-aware 远端保留剩余 TTL，TTL-unaware 远端按 L1 默认 TTL 策略回填。
- 修复 `withCache.invalidateAll()` 前缀删除可能误删同前缀手工键的问题，改为精确删除该包装函数实际写入的 key，并在失效前清理已淘汰/过期的历史 key。
- 修复 `RedisCacheAdapter` 的 `getMany/setMany/delMany` 未统一执行 key 校验的问题。
- 补齐新增缓存语义分支测试，恢复 Statements / Branches / Functions / Lines 全部 100% 覆盖率。
- 优化 CJS 构建脚本日志，隐藏预期 `TS1343` 中间诊断，避免成功构建看起来像失败。
- 移除未部署且与 README 高度重复的 Rspress website 文档站，删除相关 docs 脚本与开发依赖，降低文档同步和依赖维护成本。
- 关闭 ESM/CJS 发布构建的 source map 输出，避免 `dist/**/*.js.map` 进入 npm 包。
- 补充 README 中 `MemoryCache` / `RedisCacheAdapter` 的 TTL 查询方法说明，并将集成测试命令说明改为不依赖固定数量的表述。
